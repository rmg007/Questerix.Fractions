# K–2 Pilot Session Protocol

**Version:** 1.0 | **Duration:** ~45 minutes per child | **Audience:** Observer (Ryan or delegate)

---

## Pre-session checklist

- [ ] Device charged, app loaded to MenuScene
- [ ] Pre/post instrument printed (2 copies per child — pre and post)
- [ ] Observation rubric printed (one per session)
- [ ] Small thank-you item ready (sticker / book)
- [ ] Parent consent script ready to read

---

## Session script

```
0:00–0:02  Welcome + read parent consent script aloud (see consent-script.md)
           Get verbal confirmation from parent.

0:02–0:03  Read child assent script aloud (see consent-script.md)
           Confirm child wants to play. If no, end politely.

0:03–0:09  Administer PRE-test (pre-post-instrument.md — paper, pencil)
           Read each question aloud; child circles / marks.
           Record start and end time.

0:09–0:11  Hand child the tablet. Say only:
           "Here's the game — explore it!"
           Observer: note what they tap first, what confuses them.
           No coaching.

0:11–0:31  Free play — child plays however they like for 20 minutes.
           Observer fills observation rubric in real time.
           Neutral encouragement only: "You're doing great", "Keep going".
           No hints, no direction.

0:31–0:32  Verbal debrief (child):
           "What was your favourite part?"
           "What was tricky?"
           Transcribe verbatim answers.

0:32–0:38  Administer POST-test (same instrument, randomised question order)
           Record start and end time.

0:38–0:40  Thank the child; give thank-you item.
           Parent: brief informal debrief — "Any questions about the session?"

0:40–0:45  Observer: complete rubric fields that require post-session reflection.
           Export app session data if needed (Settings → Export or ?export=session).
```

---

## After the session

1. Transfer exported JSON to `audit/pilot-YYYY-MM-DD-NNN.md` template.
2. Score pre/post instrument; calculate delta per item.
3. Identify top 3 friction moments from rubric notes.
4. File any findings as updates to specific plan files.
5. Commit the completed audit file.
