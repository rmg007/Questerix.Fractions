## Integration Points

### 1. Dexie Persistence Layer

Hints table is part of the v3 schema and seeded during app bootstrap:

**From `src/persistence/db.ts`**:
```typescript
this.version(3).stores({
  // ... other stores ...
  hints: 'id, [questionTemplateId+order]',  // Primary key + compound index
});
```

**From `src/curriculum/seed.ts`**:
```typescript
async function seedAllStores(bundle: ParsedBundle): Promise<number> {
  await db.transaction('rw', [...tables...], async () => {
    if (bundle.hints.length > 0) {
      await db.hints.bulkPut(bundle.hints);
      total += bundle.hints.length;
    }
  });
}
```

### 2. Runtime Query Path

During gameplay, hints are fetched dynamically:

```typescript
// In interaction system (example):
const hints = await hintRepo.getForQuestion(questionTemplateId);
// Returns array of 3 HintTemplate objects, sorted by order (1→2→3)
// Runtime displays one hint per tier escalation
```

### 3. Curriculum Bundle Format

Hints exist in two locations in the pipeline output:

1. **Embedded in questionTemplates** (current v1.json structure):
   ```json
   {
     "id": "q:pt:L1:0001",
     "hints": [
       {"id": "h:pt:L1:0001:T1", ...},
       {"id": "h:pt:L1:0001:T2", ...},
       {"id": "h:pt:L1:0001:T3", ...}
     ]
   }
   ```

2. **Flattened at root level** (standalone hints.json):
   ```json
   [
     {"id": "h:pt:L1:0001:T1", "questionTemplateId": "q:pt:L1:0001", ...},
     ...
   ]
   ```

The loader (`src/curriculum/loader.ts`) currently extracts from embedded format. The standalone hints.json is available for direct seeding via `hintRepo.bulkPut()`.

## Error Handling & Resilience

### Missing hints.json

If `pipeline/output/hints.json` doesn't exist:
- Loader returns empty array (graceful degradation)
- Game continues with synthetic fallback questions
- No crash, no broken state

### Invalid Hint Records

Seeding script validates structure before insert:
- Missing required fields → logged warning
- Invalid order values → skipped
- Malformed JSON → caught by JSON parser
- Invalid schema → Dexie validation error (will not insert)

### Non-existent Template Query

Query for non-existent template:
- Returns empty array `[]`
- No error thrown
- Interaction system handles missing hints gracefully
