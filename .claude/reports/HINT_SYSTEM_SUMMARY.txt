================================================================================
TECHNICAL SPECIFICATIONS
================================================================================

ARCHITECTURE
- Modular: hints.py as standalone module
- Extensible: Per-archetype guidance in prompt
- Idempotent: Safe to re-run with same seeds
- Configurable: Batch size, model tier, retries

VALIDATION PIPELINE
1. Load curriculum (216 templates)
2. Batch into groups (~30 templates/batch)
3. Call Claude API with system + user prompts
4. Parse JSON response
5. Validate against Pydantic HintTemplate schema
6. Validate cascade (3 hints per template):
   - Word count ≤15 per hint
   - No duplicates within cascade
   - Not identical to original prompt
   - Complexity progression T1 ≤ T2 ≤ T3
7. Write output (hints.json + report.json)

OUTPUT FORMAT
- hints.json: 648 HintTemplate records (JSON array)
- hint_generation_report.json: Stats and validation results
  - generated_at: ISO timestamp
  - total_templates: 216
  - total_hints_generated: 648 (or less if validation fails)
  - validation: {passed, failed, errors}
  - stats_by_archetype: per-archetype stats
  - usage: {input_tokens, output_tokens}

COST ANALYSIS (Anthropic Haiku)
- Est. input tokens: 108,000 @ $0.80/1M = $0.09
- Est. output tokens: 129,600 @ $4.00/1M = $0.52
- TOTAL: ~$0.60 per full generation run
