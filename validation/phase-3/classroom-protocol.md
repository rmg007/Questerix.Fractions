# Classroom Protocol -- Phase 3 (25-30 Students)

Per playtest-protocol.md section 9, Cycle B is the full playtest after L1-L9 content is complete.

## 1. Setup (before students arrive, 20 min)

- Load the app on every device (25-30 devices). Pre-navigate to the Menu scene.
- Set each device display name to a pre-assigned pseudonym (label the device with a sticky note).
- Assign one observer to every 5-6 students. With 25-30 students you need 5 observers.
- Each observer has a printed stack of observation templates (one per student per session).
- Brief all observers together using teacher-prep.md before students enter.

## 2. Observation Rotation

With 5 observers for ~27 students:

| Observer | Students assigned |
|----------|------------------|
| Observer 1 | Pseudonyms A01-A05 |
| Observer 2 | Pseudonyms A06-A10 |
| Observer 3 | Pseudonyms A11-A16 |
| Observer 4 | Pseudonyms A17-A22 |
| Observer 5 | Pseudonyms A23-A27 |

During the session (15 min), each observer watches their assigned group. They do NOT circulate across groups mid-session. If a student needs help that is not a technical issue, observers note it but do not intervene.

## 3. Session Flow

| Minute | Action |
|--------|--------|
| 0-2 | Observer reads the standard script aloud to their group. Students start on prompt. |
| 2-14 | Observer watches and takes notes. No coaching. |
| 14 | Observer gives the 1-minute warning ("one more minute"). |
| 15 | Observers say "please stop playing." Click the in-app Backup My Progress button for each device. |
| 15-20 | Data export (see section 4 and data-export-checklist.md). |

## 4. Data Export at Session End

After each session, export data device-by-device before students leave. Do not skip this step.

1. Observer clicks Backup My Progress in the app.
2. Save the JSON file named: <pseudonym>_session<N>_<date>.json
3. Transfer to the teacher's machine via USB cable or local airdrop. No cloud upload.
4. Observer files their paper notes in the pseudonym folder.

Full per-device procedure is in data-export-checklist.md.

## 5. Between-Session Continuity

- Devices stay with the school or are brought back by the researcher.
- Each device retains Dexie state between sessions (do not clear the browser).
- At Session 2 start, confirm the student's progress loaded correctly before the session timer starts.

## 6. Pre-Test and Post-Test

- Pre-test: paper, administered in the classroom before Session 1 starts, scored by the researcher.
- Post-test: at least 1 day after Session 3. Same logistics, parallel form.
- Grade-appropriate forms: K students use the K-substitute for Item 7 (per pre-test.md).

## 7. Problem Escalation

| Issue | Who handles it |
|-------|---------------|
| App froze / won't load | Observer: hard-reload the page. Note the time lost. |
| Device runs out of battery | Swap with a spare device. Note in observer notes. |
| Student is distressed | Observer ends that student's session. Note "withdrew at session N." |
| Export fails | Note in checklist. Do not lose paper notes even if export fails. |
