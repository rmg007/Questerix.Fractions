# Data Export Checklist -- Phase 3

Complete this checklist for every device at the end of every session. Do not skip. Paper notes are worthless without the matching JSON export, and vice versa.

Privacy rule: no cloud upload. Transfer via USB cable or local airdrop only. Do not use Google Drive, iCloud, Dropbox, or any cloud service.

---

## Per-Device Checklist

Device pseudonym: _______________  Session #: ___  Date: _______________

### Step 1: Trigger the export in the app

[ ] The session has ended (15-minute mark reached or student withdrew).
[ ] Open the app on this device.
[ ] Navigate to Settings (gear icon, bottom-right of Menu scene).
[ ] Tap "Export Backup."
[ ] The app generates a JSON file and shows a save dialog.

### Step 2: Name and save the file

[ ] File named exactly: <pseudonym>_session<N>_<date>.json
    Example: Sparrow_session2_2026-05-14.json
[ ] File saved to the device's local Downloads folder (not to a cloud folder).

### Step 3: Transfer to researcher machine

[ ] Connect device to researcher's machine via USB cable (preferred) or use local airdrop.
    DO NOT use any cloud upload.
[ ] Confirm the file appears in the researcher's validation-data/<pseudonym>/ folder.
[ ] Open the JSON file briefly and verify it is not empty and starts with a "{" character.

### Step 4: Paper notes

[ ] Observer's paper notes for this student / session are complete (see observation-template.md).
[ ] Paper notes are labeled with the pseudonym and session number.
[ ] Paper notes are filed in the physical folder for this pseudonym.

### Step 5: Verify

[ ] Both the JSON file and paper notes are filed before the device leaves the room.
