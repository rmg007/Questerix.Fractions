## Executive Summary

* **Health verdict:** 🔴 **Structurally compromised, observably blind, presentationally broken.** The visual layer and Dexie persistence are the only two layers that materially work today. Everything in between — orchestration, learning engine, telemetry, level routing — is either disconnected, duplicated, or invisible.
* **Single greatest threat to long-term viability:** **The absent Application/Use-Case layer.** Phaser scenes (`Level01Scene.ts` ~1,200 LOC, `LevelScene.ts` analogous) have absorbed all cross-cutting orchestration: persistence dispatch, BKT updates, misconception detection, validator routing, session aggregate construction, level-unlock policy, and even hint copy. This single architectural omission is the upstream cause of (a) duplicated and divergent BKT contracts between L1 and L2+, (b) the C5 constraint breach via `localStorage` unlock state, (c) the impossibility of instrumenting validator/persistence latency (no seam exists), (d) the dynamic-`await import()` epidemic that hides dependencies from static analysis, and (e) the inability to unit-test gameplay logic without booting Phaser. Every other vulnerability in this report is downstream of this one. Fixing telemetry, portability, or UX without first introducing the Application layer will entrench the rot rather than excise it.
* **Secondary catastrophic threat:** **Production observability is `/dev/null`.** `src/lib/log.ts:47` short-circuits in `import.meta.env.PROD`, the Dexie `sessionTelemetry` table has zero writers, no `window.onerror` handler exists, and the only `unhandledrejection` listener actively *suppresses* storage errors. Once a build ships, every crash, validator exception, BKT corruption, and persistence failure evaporates with no recoverable trace.
* **Secondary acute threat (non-architectural):** Two prompt-text/validation bugs (BUG-01, BUG-02) currently make the gameplay loop impossible to complete. These are 2-minute and 30-minute fixes respectively. They do not change the architectural verdict but they block any in-browser verification of the rest.

## Dimension 1: Architecture & Portability

* **Primary Vulnerabilities:**
  * **Missing Application layer.** No `src/application/` directory exists. Use-case orchestration (`SubmitAttemptUseCase`, `OpenSessionUseCase`, `CloseSessionUseCase`, `RecordHintUseCase`, `LoadLevelTemplatesUseCase`, `ResetDeviceUseCase`, `ExportBackupUseCase`, `MarkLevelCompleteUseCase`, `GetUnlockedLevelsUseCase`) lives inside Phaser scene callbacks. Cited at `Level01Scene.ts:277-343, 437-457, 614-686, 906-1026, 1146-1174` and `LevelScene.ts:243-260, 366-420, 596-632, 634-743`.
  * **Duplicated and divergent orchestration** between `Level01Scene` and `LevelScene` — same domain logic copied with subtle drift: `LevelScene.recordAttempt` derives `skillId` as `skill.level_${levelNumber}` (line 666–717) while `Level01Scene` uses `skill.partition_halves`. The mastery store is silently bifurcating into two namespaces.
  * **Constraint C5 breach.** `MenuScene.ts:338, 353-358, 365-384` reads/writes level-unlock state to `localStorage`, then merges with Dexie. C5 forbids `localStorage` for important data; unlock progression qualifies. Quota eviction silently destroys progress.
  * **Domain layer impurity.** `src/engine/misconceptionDetectors.ts` (lines 38, 41-42, 83, 86-87, 117, 120-121, 164, 167-168, 206, 209-210) and `src/engine/selection.ts:87` directly invoke `crypto.randomUUID()`, `Date.now()`, and `Math.random()`. The `bkt.ts` header claims "no side effects" but its sibling detectors silently violate the contract — non-deterministic, untestable with property-based tests, unreplayable for forensic session reconstruction.
  * **Repository layer has no port interfaces.** All 17 files in `src/persistence/repositories/` export Dexie-typed methods directly. Engine, curriculum, and scenes all transitively pull Dexie into their dependency graph. No swap path to sqlite, HTTP-backed store, or in-memory test fake.
  * **Build-tool and host-global coupling in the logger.** `src/lib/log.ts` (lines 38, 40, 47, 114, 117, 119, 120) and `src/lib/logger.ts:6` read `import.meta.env.PROD` (Vite intrinsic), `window.location.search`, and `localStorage` — making the "centralized logger" itself a portability hazard, which is why `src/engine/calibration.ts:20` rolls its own inline logger.
  * **Dynamic `await import()` epidemic.** 18+ sites in `Level01Scene.ts` and 15+ in `LevelScene.ts` defer module resolution to dodge circular-import warnings (`Level01Scene.ts:187, 256, 282, 287, 297, 312, 313, 648, 654, 658, 805, 914, 915, 953, 954, 1007, 1011, 1149`). madge/dpdm/IDE rename refactors cannot trace these edges.
  * **Composition root not assembled.** `src/main.ts` mixes UI bootstrap, error handling, and persistence permission negotiation in one file with no DI container. A non-browser shell (Electron, Capacitor, Cloudflare Worker prerender, headless test) cannot reuse the boot sequence.
  * **Domain timestamps minted in scene code.** 25+ `Date.now()` calls across `BootScene.ts:99`, `Level01Scene.ts` (13 sites), `LevelScene.ts` (14 sites) stamp `Session.startedAt`, `Attempt.submittedAt`, `SkillMastery.lastAttemptAt`, `HintEvent.shownAt`. Domain data, scene-tier authorship.
  * **Direct DOM coupling in persistence.** `src/persistence/backup.ts:100-106` synthesises `<a download>` clicks inside the persistence layer; `src/persistence/db.ts:158-200` reads `navigator.storage` and `sessionStorage`. Persistence cannot run headlessly.
  * **Direct `db` import in SettingsScene.** `src/scenes/SettingsScene.ts:13, 256` calls `await db.delete()` from a UI button handler with no `ResetDeviceUseCase` audit/guard.

* **Required Deep-Dive:**
  1. **`src/scenes/Level01Scene.ts` and `src/scenes/LevelScene.ts`** — extract use cases, target shrink from ~1,200 to ~400 LOC each; sunset `Level01Scene.ts` once parity is reached (Decision D-4 in master plan). **Highest leverage; gates everything below.**
  2. **`src/scenes/MenuScene.ts`** (lines 338-384, 765+) — extract `ProgressionRepository` (extend existing `progressionStat.ts`), `LevelUnlockPolicy` pure function in `src/engine/`, and `GetUnlockedLevelsUseCase`. Migrate `localStorage` keys on first boot; close C5 breach.
  3. **`src/engine/`** — introduce `src/engine/ports.ts` with `Clock`, `IdGenerator`, `Rng`, `Logger` interfaces; thread `DetectorContext` through every detector; inject from composition root. Add lint rule forbidding `crypto`, `Date`, `Math.random`, `window`, `document`, `console`, `localStorage`, `fetch` in `src/engine/**` and `src/validators/**`.
  4. **`src/persistence/repositories/*` (all 17)** — define sibling `XRepository` interfaces in `src/persistence/ports/`; move concrete Dexie implementations to `src/persistence/dexie/`. Compose at boot, not at module-import time.
  5. **`src/curriculum/loader.ts`** — split into `parseCurriculumBundle(source: BundleSource)` plus `HttpBundleSource`, `EmbeddedBundleSource`, and `FallbackBundleSource(primary, fallback)` adapters in `src/lib/adapters/`. Removes hardcoded `fetch('/curriculum/v1.json')` and the silent Vite JSON-import dependency.
  6. **`src/main.ts`** — promote to thin composition root; assemble `Composition = { clock, ids, rng, logger, bundleSource, repos, sink, permission, viewport }` and pass via `scene.data` registry.

## Dimension 2: Telemetry & Observability

* **Critical Blind Spots:**
  * **No production error sink whatsoever.** `src/lib/log.ts:47` returns `false` from `isEnabled` whenever `import.meta.env.PROD` is true. Every `log.error(...)`, `log.warn(...)` evaporates in shipped builds. `src/lib/logger.ts:6-20` mirrors this — only `console.error` survives prod, with no remote shipper.
  * **No `window.onerror` handler.** Verified by `Grep` — zero matches for `window.onerror|addEventListener\s*\(\s*['"]error`. Synchronous uncaught exceptions in Phaser scene callbacks (the most common form) are completely unrecoverable. The only existing `unhandledrejection` listener at `src/main.ts:7-14` actively *suppresses* storage errors via `event.preventDefault()` without persisting them.
  * **`sessionTelemetry` Dexie table is a dead-letter store.** Schema declares it at `db.ts:50, 67, 80, 105, 136`; only reader is `backup.ts:61`; no writer exists in the entire `src/` tree. The capture object is constructed at `Level01Scene.ts:333` and `LevelScene.ts:623` (viewport, FPS targets) and immediately discarded — wire is cut between schema, capture, and persistence.
  * **No correlation IDs across the attempt cascade.** `recordAttempt` triggers ≥6 downstream operations (validator dispatch, attempt write, detector run, flag write × N, BKT update, mastery write) that share no `traceId`. Forensic reconstruction of a corrupted student record requires reasoning about wall-clock proximity in scrollback.
  * **No execution-time measurement** of (a) any of 17 Dexie repositories, (b) validator dispatch (`Level01Scene.ts:652`, `LevelScene.ts:393`), (c) curriculum bundle fetch (`loader.ts:138`), (d) seed transactions (`seed.ts`), (e) boot-phase durations. Every latency regression is invisible.
  * **No structured logging.** Emit at `log.ts:85-91` is `console[fn](prefix, %cStyle, %cReset, data)` — pretty for devtools, unparseable for any log aggregator. Zero JSON serialisation, zero stable schema.
  * **No frame-budget instrumentation.** Zero `performance.now`, zero `performance.mark|measure|memory`, zero `PerformanceObserver`, zero `game.loop.actualFps` reads (verified by Grep). Janks > 50 ms on the partition-drag path are silent on the primary K-2 device (iPad).
  * **No quota or memory telemetry.** `navigator.storage.estimate()` is called only inside `db.ts:158-200`'s grant negotiation, never to monitor headroom. `navigator.deviceMemory` and `performance.memory` are unread. The very failure mode `db.ts` was written to *prevent* (`QuotaExceededError`) routes back through the suppressing `main.ts` rejection handler.
  * **Errors swallowed without any sink.** `src/lib/preferences.ts:28-34` discards exceptions with no log call at all. `src/scenes/MenuScene.ts:789` silently swallows font-load failures. `src/scenes/Level01Scene.ts:997-1024` reduces three nested catches to `String(err)` (drops stack), no originating IDs captured.
  * **No build/release version stamp** on any log record. Even if logs reached an aggregator, no `app.version` / `git.sha` field exists for deploy correlation.
  * **Two parallel logger implementations plus a third inlined.** `lib/log.ts`, `lib/logger.ts`, and `engine/calibration.ts:20`'s ad-hoc inline. Three substrates to retrofit; engine team correctly avoids the central one because it taints purity.

* **Instrumentation Priority:**
  * **Step 1 (immediate, before any tooling):** Rewrite `src/lib/log.ts` as a `Logger` factory returning `{ debug, info, warn, error: (event, attrs) => void }` with a sink array. Sinks: `ConsoleSink` (dev only), `IndexedDBSink` (writes to a new `telemetryEvents` table at Dexie schema v4). Production builds keep the IndexedDB sink active. **This single change removes the production silence and creates the durable buffer that any future remote shipper can drain.** Mirror in `src/lib/logger.ts` or delete the duplicate. Eliminate the inline logger in `engine/calibration.ts` simultaneously.
  * **Step 2 (same release):** Install paired `window.addEventListener('error', ...)` and rewritten `unhandledrejection` handler in `src/main.ts` that route through the new `Logger.error`. Move storage-error suppression *into* the logger as a `severity: 'warn'` tag rather than dropping the record.
  * **Step 3 (next release):** Wire the `sessionTelemetry` writer at the end of `openSession()` in both scenes — capture viewport, `navigator.userAgent`, `navigator.deviceMemory`, `screen.devicePixelRatio`, `prefersReducedMotion` snapshot. Connect the cut wire.
  * **Step 4 (post-Application-layer extraction):** Wrap repositories and validators with `performance.now()` brackets at the application-service boundary (the seam that does not exist today). Emit `db.op.duration_ms`, `validator.dispatch.duration_ms`, `boot.phase.duration_ms` histograms into the IndexedDB ring buffer.
  * **Step 5 (release-grade):** Adopt OpenTelemetry-JS (`@opentelemetry/sdk-trace-web`, `@opentelemetry/sdk-metrics`, OTLP/HTTP exporter via `navigator.sendBeacon`) and `@sentry/browser` for error aggregation. Stamp every record with `import.meta.env.VITE_GIT_SHA` set by Vite from CI.
  * **Step 6 (privacy gate):** Add `DeviceMeta.preferences.telemetryConsent` flag honoured in the buffered exporter — IndexedDB sink always writes (in-app debugging), network flush is opt-in. Reflect in `public/privacy.html`.

## Dimension 3: Dependency Longevity

* **Volatile Dependencies:**
  * **`workbox-window` declared twice** — appears in both `dependencies` (line 62) and `devDependencies` (line 46) of `package.json`. Resolves to a single version today, but ships ambiguous intent and risks dual-version drift on next bump. **Action:** drop from `devDependencies`; runtime concern.
  * **`nanoid` is a hidden runtime dependency.** `Level01Scene.ts:313, 915` and `LevelScene.ts` analogous lines call `await import('nanoid').catch(...)` with a runtime-resolved fallback. **`nanoid` is not declared in `package.json` at all.** Either the catch fallback always fires (silent dead code path) or the Vite build chokes once tree-shaking decides differently. Replace with the `IdGenerator` port (`crypto.randomUUID()` in the browser adapter); remove the dynamic import; never add `nanoid`.
  * **`@tailwindcss/vite` 4.x + `tailwindcss` 4.x** — Tailwind v4 is the most recently released major and the v4 Vite plugin is the official migration path. Stable enough to keep, but the codebase uses very little of Tailwind's surface; styles live mostly in Phaser scenes (canvas-rendered). **Action:** audit actual Tailwind class usage in `src/styles/index.css` and the small DOM components (`AccessibilityAnnouncer`, `SkipLink`, `PreferenceToggle`, `SettingsScene` overlay). If usage is < 5 directives, replace with hand-written CSS and drop both packages — eliminates a build-tool plugin and a moving-target framework.
  * **`vite-plugin-pwa` 0.21.x + `workbox-build` / `workbox-window` 7.4** — PWA tooling. `vite-plugin-pwa` is community-maintained, sub-1.0, and has historically broken on every Vite major. Vite 8 is current here. Workbox itself is Google-maintained but the surface area used (precache + simple runtime caching) can be hand-rolled in ~80 lines of service-worker code. **Action:** measure actual Workbox features in use; if it is just precache + offline fallback, replace with a hand-written service worker registered directly. If complex caching strategies justify staying, freeze on the current `vite-plugin-pwa` version and add a smoke test.
  * **`fast-check` 3.23** — declared but property-based tests do not yet exist for the engine (the natural consumer). **Action:** either land the engine property tests in Sprint 5 (justifies retention) or remove the dependency until needed.
  * **`fake-indexeddb` 6.2** — necessary for Vitest persistence tests; keep.
  * **`jsdom` 29** — Vitest default DOM; keep.
  * **`@types/node` 25.6** — used only by build scripts; reasonable.
  * **`vitest` 1.0.0** — pinned to a 1.x baseline. Vitest is currently on 2.x with active feature work; staying on 1.0 is defensible (tests pass) but accept the stagnation. **Action:** pin or budget a single migration sprint; do not let it drift.
  * **`typescript` 6.0.3** — current major track. Compatible with Vite 8. Keep.
  * **`eslint` 8.56 + `@typescript-eslint` 7.x** — eslint 9.x is now flat-config; the project is one ecosystem-shift behind. Defensible to stay until the next quiet sprint.
  * **`@playwright/test` 1.59** — current. Keep.
  * **`@axe-core/playwright` + `axe-core` 4.10** — boring, valuable. Keep.
  * **Phaser 4.0.0** — load-bearing; not a candidate for removal under the locked C4 constraint. Note: 4.0 is the most recent major — if it proves unstable, consider freezing to a known-good minor and cherry-picking patches rather than tracking 4.x main.
  * **Dexie 4.4.2** — boring, working, well-maintained. Keep.

* **Dead Code:**
  * **`src/scenes/Level01Scene.ts` (~1,200 LOC).** The generic `LevelScene.ts` accepts `levelNumber: 1` and is functionally a superset. Master-plan Decision D-4 explicitly asks whether to sunset. **Once Application-layer extraction lands, delete `Level01Scene.ts` entirely.** Eliminates the divergent BKT contract and ~1,200 LOC of duplicated orchestration.
  * **Hardcoded `QUESTIONS` array at `Level01Scene.ts:69-110`** — fallback templates baked into the presentation layer. Resolve via `CurriculumGateway` port; delete.
  * **Per-archetype hint copy table at `LevelScene.ts:495-553`** — 60 lines of copy data inside a Phaser scene method. Move to `src/curriculum/hints/` as JSON keyed by `(archetype, tier)`; resolve via `HintCopyRepository`.
  * **Five copies of `prefersReducedMotion` checks** at `Level01Scene.ts:1180`, `LevelScene.ts:873`, `MenuScene.ts:767`, `PreloadScene.ts:130`, plus the SettingsScene toggle. Extract `src/lib/accessibility.ts`.
  * **`src/scenes/MenuScene.markLevelComplete` static method** — presentation class acting as domain-state writer. Delete after `MarkLevelCompleteUseCase` exists.
  * **Inline `logger` object at `src/engine/calibration.ts:20`** — workaround for a bad central logger. Delete once `Logger` port lands.
  * **`logger.ts` vs `log.ts`** — two parallel logging substrates; one must die. `log.ts` has the richer category surface; `logger.ts` is a thin level wrapper. **Action:** keep `log.ts` (rewritten as factory + sinks per Dimension 2 Step 1); delete `logger.ts` and migrate the ~5 callers.
  * **`PLANS/phase-3-*.md` planning documents** (cited in `INDEX.md` lines 75-138) — superseded by `master-plan-2026-04-26.md` + `architecture-review-2026-04-27.md`. Move to `PLANS/archive/` or delete to reduce planning-suite navigation cost.
  * **`mockup-sandbox/`, `attached_assets/`, `screenshots/`** at repo root — quarantined inherited artifacts (per memory `project_inherited_artifacts.md`). Audit and either consolidate into `PLANS/screenshots/` (already referenced in `qa-visual-report-2026-04-27.md`) or delete.
  * **`src/scenes/Level01Scene.ts` dynamic-import fallback for `nanoid`** (lines 313, 915) — both a code-smell and a hidden dependency. Delete; route through `IdGenerator` port.
  * **Suspected:** validators with no live caller — audit `src/validators/` against `validatorRegistry.ts` for orphans.

## Dimension 4: UX & Cognitive Friction

* **Friction Points:**
  * **BUG-01 — Prompt-mechanic incoherence.** Student opens Level 1 ("Halves") and reads *"Which shape has 1/3 shaded?"* while the screen shows a draggable line on a rectangle. Three simultaneous violations of the K-2 cognitive contract: wrong archetype (`identify` vs `partition`), wrong fraction (1/3 vs 1/2), wrong action verb ("which shape" vs "split"). A 6-year-old has no recovery path. `Level01Scene.ts` template filter — fix is one line. **The single highest-friction defect in the system.**
  * **BUG-02 — Validation never accepts a correct answer.** "Check ✓" produces no feedback, no progress increment, no animation. Student sits at 0/5 indefinitely with no diagnostic signal. `Level01Scene.onSubmit()` `handlePos` plumbing or snap tolerance.
  * **BUG-04 — Hint ladder stuck on Tier 1.** Repeated hint presses replay the same Tier-1 message; Tier 2 (visual midpoint overlay) and Tier 3 (worked example) never fire. Removes the only available scaffold for a struggling student.
  * **G-C3 — No UI route to Level 2-9.** The adventure map is decorative. Master-plan Section 5.1 confirms exactly three tappable stations: Play!, Continue, Settings. A student who finishes L1 has nowhere to go. Direct cause of "couldn't go till the last level" feedback.
  * **G-C7 — "Keep going" loops Level 1.** `LevelScene.showSessionComplete()`'s "Keep going ▶" button calls `loadQuestion(questionIndex + 1)` which increments within the same level rather than advancing `levelNumber`. Student completes "Halves" and is immediately handed another halves question — invisible regression.
  * **G-UX3 — TTS built but never called.** `TTSService` (Web Speech API, K-2-tuned 0.95× rate) exists; `PreferenceToggle` works in `SettingsScene`; the prompt is never spoken. K-2 students with developing reading fluency cannot independently start a session that requires reading the prompt. Critical accessibility regression for the target persona.
  * **Settings-gear ambiguity.** Earlier QA flagged BUG-05 (settings gear routed to Level 1); architecture review reclassified as IDE-preview artifact since `MenuScene` calls `scene.launch('SettingsScene')`. **Cognitive cost:** until verified in a real browser, the team carries a phantom risk and cannot trust the menu's primary affordance.
  * **No feedback verified for K-2 emotional appropriateness.** `EXACT → green + "Correct! Great work."`, `CLOSE → amber + "Almost! Try a tiny adjustment."`, `WRONG → shake + "Not quite — try again."` are all blocked by BUG-02 and have never been seen by a target-age proxy. Risk: animations or copy land flat or scary.
  * **Hint copy is hardcoded in scene methods** (`LevelScene.ts:495-553`) — ineligible for i18n, A/B testing, or non-engineer authoring. Every wording change needs a code release.
  * **No frame-budget signal on the partition-drag path.** Combined with the iPad/touch-untested status, the core mechanic's fluidity on the primary device is unknown.
  * **Adventure-map nodes are decorative** — visual affordance suggests tappability; no handler attached. Standard cognitive-load violation: the UI lies about what's interactive.
  * **Settings-scene reachability and reset-device gesture** are not yet retested; reset is a 2-step confirm wrapping a raw `db.delete()` with no audit log — an unnecessarily heavy and silent destruction.
  * **`SettingsScene` opens via `scene.launch` overlay** — correct in code, but if the overlay doesn't visually communicate "this is overlaid on the menu, not a new screen," students may lose orientation. Pending real-browser verification.

## Phased Execution Strategy

* **Phase 1 — Stabilise the present, illuminate the dark, then unblock the loop (Sprint 0 + half of Sprint 0.5):**
  1. **Same day:** Fix BUG-01 (template filter to `archetype === 'partition'`, ~10 min) and verify in a real Chrome tab. Capture the failing path of BUG-02 with the existing `?log=DRAG,VALID,Q` filter, then fix `handlePos` plumbing (~30 min). Fix BUG-04 hint-tier counter (~15 min). Retest the settings-gear (BUG-05) in a real browser to either close or escalate.
  2. **Same week:** Rewrite `src/lib/log.ts` as a `Logger` factory + sink array (`ConsoleSink` dev-only, `IndexedDBSink` always-on). Bump Dexie to schema v4 with a `telemetryEvents: '++id, kind, severity, ts, traceId, [traceId+ts]'` table and `telemetryEventRepo`. Install paired `window.addEventListener('error', ...)` + rewritten `unhandledrejection` in `src/main.ts` that route through the new logger. **Removes the production-silence catastrophic blind spot before any further code lands.** Migrate the ~5 callers of `lib/logger.ts` and the inline logger in `engine/calibration.ts`; delete both. Wire the `sessionTelemetry` writer at session open. Drop the duplicate `workbox-window` from `devDependencies`. Remove the `nanoid` dynamic import (route through a temporary inline `crypto.randomUUID()` until the `IdGenerator` port lands in Phase 2).
  3. **Phase 1 exit:** Student completes a 5-question session in a real browser; production builds emit structured errors to IndexedDB; no scene calls a deprecated logger; package.json declares no hidden runtime deps.

* **Phase 2 — Introduce the Application layer; close C5 and the BKT divergence (Sprint 1 + Sprint 2):**
  1. **Engine-purity ports first:** Add `src/engine/ports.ts` with `Clock`, `IdGenerator`, `Rng`, `Logger` interfaces. Refactor `misconceptionDetectors.ts`, `selection.ts`, and `calibration.ts` to receive ports. Add lint rule forbidding `crypto`, `Date`, `Math.random`, `window`, `document`, `console`, `localStorage`, `fetch` in `src/engine/**` and `src/validators/**`. Provides deterministic detectors and unblocks property-based engine tests with `fast-check`.
  2. **Stand up `src/application/`** with `SubmitAttemptUseCase`, `OpenSessionUseCase`, `CloseSessionUseCase`, `LoadLevelTemplatesUseCase`, `RecordHintUseCase`, `MarkLevelCompleteUseCase`, `GetUnlockedLevelsUseCase`, `ResetDeviceUseCase`, `ExportBackupUseCase`. Move the ~600 LOC of orchestration out of both scene files. Introduce `SkillIdResolver` (single mastery namespace), `SessionFactory`, `SessionSummaryCalculator`, `LevelUnlockPolicy`, `ValidationService`. Wire BKT (`updateMastery`) and detectors (`runAllDetectors`) into `SubmitAttemptUseCase`. **Closes G-E1 through G-E5 in one structural move and silences the divergence drift.**
  3. **Replace MenuScene `localStorage` unlock state** with `ProgressionRepository` writes via `MarkLevelCompleteUseCase`. One-time migration copies existing `localStorage` keys, then deletes them. **Closes the C5 breach.** Make adventure-map nodes tappable through `GetUnlockedLevelsUseCase`. Fix "Keep going" to advance `levelNumber` (G-C7).
  4. **Sunset `Level01Scene.ts`.** Once `LevelScene.ts` consumes the use cases and reaches parity, delete `Level01Scene.ts` and its hardcoded `QUESTIONS` array. Replace dynamic `await import()` calls with eager top-of-file imports; add the ESLint `no-restricted-syntax` rule that forbids dynamic imports of `src/persistence/**` and `src/engine/**` from `src/scenes/**`.
  5. **Application-layer instrumentation:** at the use-case seam (which now exists), wrap repository and validator dispatch with `performance.now()` brackets emitting to the `Logger`. **Phase 2 exit:** mastery state visible in IndexedDB after 5 questions; Level 2 is reachable from the menu; no scene imports a Dexie repository directly; no detector calls `Date.now()` or `crypto`; latency histograms accumulate in `telemetryEvents`.

* **Phase 3 — Distil the interface, harden the perimeter, ship (Sprint 3 + Sprint 4 + Sprint 5):**
  1. **Cognitive-load distillation:** Activate TTS at question load (G-UX3, G-UX8). Move hardcoded hint copy to `src/curriculum/hints/` JSON. Extract `src/lib/accessibility.ts` for the five duplicated `prefersReducedMotion` checks. Playtest feedback animations with a target-age proxy. Author L2-L9 templates against the now-trustworthy use cases. Verify session-complete card communicates progression clearly.
  2. **Portability hardening:** Repository port interfaces (`src/persistence/ports/`) + `src/persistence/dexie/` concrete implementations. `BundleSource` adapters in `src/lib/adapters/`. `BackupSink` adapters. `Viewport` port. Promote `src/main.ts` to thin composition root assembling the full DI container. Removes every remaining host-global from the domain.
  3. **Observability graduation:** Adopt OpenTelemetry-JS + `@sentry/browser`; replace the `IndexedDBSink` with a `BufferedExporter` that drains via `navigator.sendBeacon` on `pagehide`. Stamp every record with `import.meta.env.VITE_GIT_SHA` injected by Vite from CI. Add `DeviceMeta.preferences.telemetryConsent` privacy gate.
  4. **Production gates:** iPad Safari touch-drag verification (G-OPS2 — primary K-2 device, currently untested). Playwright happy-path E2E (`tests/e2e/`). Expand BKT and validator unit tests with `fast-check`. Hand-rolled service worker (or freeze `vite-plugin-pwa`). Lighthouse PWA audit. Cloudflare Pages deploy via existing `wrangler.toml`.
